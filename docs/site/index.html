Why ferris-db?

Built from the ground up in Rust for performance, safety, and compatibility

⚡

True Multi-threading

Concurrent command execution with sharded storage using DashMap. Key-level locking ensures maximum parallelism.

🔒

Memory Safe

Zero unsafe code blocks. Memory safety guaranteed by the Rust compiler. No buffer overflows, no data races.

🔄

Drop-in Replacement

Works with redis-cli, redis-py, Jedis, ioredis, and all standard Redis clients. Just change the port.

📦

Rich Data Types

Strings, Lists, Hashes, Sets, Sorted Sets, HyperLogLog, Geo, and more. Full support for complex data structures.

💬

Pub/Sub & Transactions

Real-time messaging with pattern subscriptions. MULTI/EXEC transactions with WATCH for optimistic locking.

💾

AOF Persistence

Append-only file with background rewriting. Millisecond-precision TTL with lazy and active expiry.

Works with Any Redis Client

Just change the port and you're ready to go

# Connect to ferris-db
$ redis-cli -p 6380

# String operations
127.0.0.1:6380> SET hello "world"
OK
127.0.0.1:6380> GET hello
"world"

# Atomic increment
127.0.0.1:6380> INCR counter
(integer) 1

# Sorted sets
127.0.0.1:6380> ZADD leaderboard 100 "alice" 200 "bob"
(integer) 2
127.0.0.1:6380> ZRANGE leaderboard 0 -1 WITHSCORES
1) "alice"
2) "100"
3) "bob"
4) "200"

Quick Start

Get up and running in under a minute

1

Clone and Build

$ git clone https://github.com/ferrishq/ferris-db.git
$ cd ferris-db
$ cargo build --release
2

Start the Server

$ ./target/release/ferris-db --port 6380
3

Connect with Any Redis Client

$ redis-cli -p 6380
127.0.0.1:6380> PING
PONG
